The Problem

Async propagation
Arrival order = topology leak
Fast peers = relay heuristic signal
Co-location signal = shared /24
Feature bits = risk surface
Lightning gossip propagates asynchronously. Who receives a message first reveals network topology. Peers that consistently arrive in the top 5% are a relay-timing heuristic signal, not direct surveillance proof. We treat shared /24 prefixes as co-location signals: useful infrastructure hints, but not proof of common operator control. And feature bits in node_announcement expose implementation fingerprint and risk surface.

Our Approach

24h exported data
Wavefronts
Fast-relay heuristics
Co-location signals (/24)
Feature fingerprint
We use exported 24-hour data from the gossip observer project — we don't run a node. We build wavefronts: per-message arrival times. Identify fast-relay heuristics using top-5% timing thresholds. Build co-location signals from shared /24 prefixes in node_announcement clearnet addresses. Parse feature bits for protocol-level risk exposure.
